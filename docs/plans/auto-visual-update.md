## 键盘按键命中区域自适应支持

  ### 摘要

  在不改变现有键盘视觉排版的前提下，为全键盘和紧凑键盘模式增加“自适应命中区域”。实现方式是动态扩展每个可见按键的
  invisible hitbox，而不是放大按键本身。目标是让窄键、边缘键、紧凑模式下的控制键更容易点中，同时保持现有点击、长按、上
  滑、删除连发等行为不变。

  ### 关键实现

  - 新增一个按行工作的复合触摸代理层，挂在每个键盘行容器上，而不是改现有每个键的点击逻辑。
      - 代理层负责维护“扩展后的命中矩形 -> 目标 View”的映射。
      - ACTION_DOWN 先按扩展后的矩形选中目标键，后续 MOVE/UP/CANCEL 固定发给同一个键，保证长按、删除连发、上滑符号这些现
        有手势不被打断。
  - 命中区域自适应规则固定为：
      - 最小目标尺寸按 48dp x 48dp 计算。
      - 视觉尺寸已经达到阈值的键不扩展。
      - 宽度不足的键优先向左右扩展；高度不足的键只在本行上下安全带内扩展，不跨整行去抢相邻行的点击。
      - 相邻键扩展后如果会重叠，按两个键中心点的中线切分，避免一个区域同时命中两个键。
      - 行首、行尾键可向外侧边缘补足，但不超出整行容器边界。
  - 作用范围固定为全部模式：
      - FULL
      - COMPACT_ZH
      - COMPACT_EN
  - 覆盖对象固定为键盘主键区和 FN 行的所有可见键，不包含宏栏和候选词栏。
  - 现有按键行为保持原样：
      - 普通点击
      - 字母键上滑输入替代符号
      - 长按替代符号
      - Backspace/Del 长按连续删除
      - Backspace/Del 上滑清空
      - MIC、模式键、方向键、控制键的现有 OnClickListener / OnTouchListener 不重写业务逻辑，只改变更早的命中分发

  ### 实现落点

  - 在输入服务侧增加一个“命中区域重建”流程，集中在布局变化后调用。
      - 初次 onCreateInputView() 完成后执行一次
      - 每次 updateKeyboardLayoutUi() 执行完可见性、权重、行高调整后重新计算一次
      - 紧凑/全键盘切换、中英切换、行高变化后都重新生效
  - 新增一个独立的辅助类处理复合 TouchDelegate，职责只做：
      - 收集一行内当前可见子键的实际矩形
      - 生成扩展后的命中矩形
      - 将触摸事件转发给正确的子 View
  - 不新增设置项。该能力默认启用。

  ### 测试方案

  - 命中准确性：
      - 紧凑模式下窄标点键、边缘功能键、方向键比当前更容易点中
      - 全键盘模式下命中改善，但不会明显串到相邻键
  - 交互回归：
      - 字母键点击、长按、上滑仍按原规则工作
      - Backspace/Del 长按连续删除和上滑清空不回归
      - MIC 点击、录音态闪烁、模式切换键不受影响
  - 布局联动：
      - 切换 FULL / COMPACT_ZH / COMPACT_EN 后命中区域会同步更新
      - 隐藏的键不会保留旧的命中区域
  - 边界场景：
      - 行首、行尾键在屏幕边缘仍能稳定点中
      - 上下相邻两行之间不会出现跨行误触
      - 横屏、不同键高配置下仍遵守最小目标尺寸和中线切分规则

  ### 默认假设

  - “自适应支持”解释为“视觉不变，只扩命中区域”。
  - 默认对全部键盘模式生效，不单独加开关。
  - 采用 48dp 作为最小触摸目标尺寸。
  - 宏栏和候选栏不纳入这次范围，只处理真正的键盘按键。
